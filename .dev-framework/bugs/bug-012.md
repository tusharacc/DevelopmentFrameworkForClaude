# BUG-012: Library sort heuristic "shortest name first" may omit important dependencies

**ID:** BUG-012
**Severity:** low
**Status:** open
**Workspace:** brownfield-repo-analyzer

## Description

Step 2g of `skills/init-brownfield/SKILL.md` instructs:
> "libraries (top 10 by name length, shortest first)"

Sorting by shortest name first is a poor proxy for importance. It will surface short-named packages (e.g., `fs`, `os`, `rx`, `joi`) over important long-named ones (e.g., `@tanstack/react-query`, `@prisma/client`). The intent is presumably to show the most important libraries, not the ones with the shortest names.

## Fix

Replace the heuristic with: list all direct dependencies (not devDependencies) up to 10, in the order they appear in `package.json`. If more than 10, list the first 10 and note the count.

## Impact

Low — affects only the display quality of the Key libs row in the discovery summary. Users can correct this at the confirmation step (Step 4).

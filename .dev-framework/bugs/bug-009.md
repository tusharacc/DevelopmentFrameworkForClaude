## Description
The fix for BUG-008 hardcoded absolute paths in `hooks/hooks.json`:
```json
"command": "bash /Users/tusharsaurabh/Documents/Projects/AI/DevelopmentFrameworkForClaude/hooks/check-phase.sh"
```
If the repo is cloned to a different path or used by another developer, the hook will fail with the same "No such file or directory" error that BUG-008 originally caused.

## Expected Behaviour
Hook paths should resolve dynamically regardless of where the repo is cloned.

## Fix Ideas
- Use a wrapper script that resolves the hook path relative to `$BASH_SOURCE` (e.g. `$(dirname "$BASH_SOURCE")/check-phase.sh`)
- Or configure hooks via `.claude/settings.json` using a path computed at install time

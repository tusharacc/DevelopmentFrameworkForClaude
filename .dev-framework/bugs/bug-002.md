# BUG-002: No maximum loop count for reviewer → developer review cycles

**Severity**: low
**Status**: open
**Created**: 2026-04-11T00:00:00Z
**Source**: Code review of BUG-001 fix (R6)

## Description
The hand-off skill allows the reviewer to loop back to developer indefinitely if high/medium issues are repeatedly found. There is no guard to detect and break an infinite review cycle.

## Expected Behavior
After a configurable number of reviewer → developer loops (e.g. 3), the framework should flag the workspace as "blocked" and notify the user to intervene manually.

## Actual Behavior
No maximum is enforced — loop can repeat forever.

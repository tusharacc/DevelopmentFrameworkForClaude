## Description
`hooks/check-phase.sh` and `hooks/check-bash-phase.sh` use `grep -q "^\.dev-framework/"` to allowlist framework files, but the Write and Edit tools always pass absolute paths (e.g. `/Users/foo/.../DevelopmentFrameworkForClaude/.dev-framework/checkpoint.md`). The `^` anchor never matches, so the allowlist is bypassed and the hook blocks legitimate writes to `.dev-framework/` and `hooks/` when the workspace is not in the `developer` phase.

## Steps to Reproduce
1. Have an active workspace in any non-developer phase (e.g. `complete`).
2. Attempt to write any file under `.dev-framework/` using the Write tool.
3. Hook blocks the write even though it should be allowed.

## Expected Behaviour
Writes to `.dev-framework/`, `hooks/`, `skills/`, `agents/`, `CLAUDE.md` are always allowed regardless of current phase.

## Fix
Replace `^\.dev-framework/` with `(^|/)\.dev-framework/` (and similar for other allowlist patterns) so absolute paths are matched correctly.

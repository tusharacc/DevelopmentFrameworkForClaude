# BUG-013: Commands template placeholder behaviour is ambiguous when value not detected

**ID:** BUG-013
**Severity:** low
**Status:** open
**Workspace:** brownfield-repo-analyzer

## Description

The Case A CLAUDE.md template in Step 5 shows:
```
- **Build:** `[command]` *(not detected)*
```

The rules say:
- "Replace `*(not detected)*` with the actual value when known"
- "keep the literal text `not detected` when unknown"

But the template still includes `[command]` as a placeholder alongside `*(not detected)*`. It is unclear whether the final output when a command is not found should be:
- `` - **Build:** `not detected` `` (replacing both `[command]` and `*(not detected)*`)
- `- **Build:** *(not detected)*` (removing `[command]` but keeping the annotation)
- `- **Build:** [command] *(not detected)*` (leaving both as-is)

## Fix

Rewrite the template rules to be unambiguous:
- If command detected: `- **Build:** \`npm run build\``
- If command not detected: `- **Build:** not detected`

Remove the `*(not detected)*` inline annotation style entirely — it creates formatting noise. The plain text `not detected` is greppable and clear.

## Impact

Low — affects only the formatting consistency of generated CLAUDE.md files. Does not affect correctness of the analysis.

# BUG-018 — No default .code-quality-ignore for rules/hooks files (self-referential false positives)

**Severity:** low
**Status:** open
**Workspace:** code-quality-agents
**Created:** 2026-05-02

## Description

`skills/code-quality/rules/secret-patterns.md` and `skills/code-quality/hooks/pre-commit.sh` contain regex pattern strings that match the scanner's own detection patterns (e.g., the PRIVATE_KEY_PEM regex string `-----BEGIN ... PRIVATE KEY-----` appears in both files as documentation/code). Every time the secret-detection agent scans a branch that includes changes to these files, it generates false positives.

No default `.code-quality-ignore` file is committed with the feature, so false positives persist until manually suppressed.

## Fix

Commit a `.code-quality-ignore` file at repo root with entries suppressing the rules and hooks directories:
```
# Justification: rule definition files contain regex patterns that look like secrets — they are pattern strings, not real credentials
skills/code-quality/rules/
skills/code-quality/hooks/pre-commit.sh
```

# BUG-019 — 4 patterns in secret-patterns.md absent from pre-commit hook

**Severity:** low
**Status:** open
**Workspace:** code-quality-agents
**Created:** 2026-05-02

## Description

`skills/code-quality/rules/secret-patterns.md` documents 16 detection patterns. The hook `skills/code-quality/hooks/pre-commit.sh` only implements 10 of them. The following 4 are missing from the hook's `PATTERN_NAMES`/`PATTERN_REGEXES` arrays and are never checked at commit time:

- `AWS_SECRET_ACCESS_KEY`
- `GCP_SERVICE_ACCOUNT_KEY`
- `AZURE_CLIENT_SECRET`
- `TWILIO_KEY`
- `SENDGRID_KEY` (also absent, making it 5 missing)

The gap is undocumented — developers would assume all 16 patterns are enforced at commit time.

## Fix

Either add the missing patterns to the hook (with their ERE-compatible regexes), or add a comment in `secret-patterns.md` explicitly noting which patterns are LLM-agent-only (review mode) vs. hook-enforced.

# BUG-015: README first paragraph capture may pull badge/shield lines

**ID:** BUG-015
**Severity:** low
**Status:** open
**Workspace:** brownfield-repo-analyzer

## Description

Step 2h instructs: "Extract: the first non-empty paragraph after the title (this becomes the Project Overview)."

Many real-world READMEs place badge/shield lines (e.g., `[![Build](https://...)](https://...)`) immediately after the title, before any prose description. The instruction "first non-empty paragraph after the title" would capture badge lines as the Project Overview, resulting in CLAUDE.md containing a block of markdown image syntax instead of a human-readable description.

## Fix

Add a clarification to Step 2h:
> "Skip lines that start with `[![` or `![` (badge/image lines). Extract the first non-empty paragraph that consists of regular prose text."

## Impact

Low — affects only the Project Overview section of the generated CLAUDE.md. Users can correct this at the confirmation step (Step 4).

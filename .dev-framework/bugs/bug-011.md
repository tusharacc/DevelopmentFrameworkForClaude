# BUG-011: find -o without parentheses in Step 2f CI detection

**ID:** BUG-011
**Severity:** low
**Status:** open
**Workspace:** brownfield-repo-analyzer

## Description

Step 2f of `skills/init-brownfield/SKILL.md` contains:

```bash
find . -maxdepth 2 -name "Jenkinsfile" -o -name ".circleci" -o -name ".travis.yml" 2>/dev/null | head -5
```

Due to `find` operator precedence, `-maxdepth 2` only applies to the first `-name` clause (`Jenkinsfile`). The `-o` (OR) alternatives are evaluated independently without the depth constraint, so `.circleci` and `.travis.yml` could be matched anywhere in the tree.

## Expected Behaviour

All three filename patterns should be constrained to `-maxdepth 2`.

## Fix

Wrap the name alternatives in parentheses:
```bash
find . -maxdepth 2 \( -name "Jenkinsfile" -o -name ".circleci" -o -name ".travis.yml" \) 2>/dev/null | head -5
```

## Impact

Low — CI detection is informational only and does not affect what is written to CLAUDE.md. The bug could cause false matches deep in the tree (e.g., inside vendored copies), slightly polluting the CI/CD row of the discovery summary.

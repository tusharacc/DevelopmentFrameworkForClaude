# BUG-014: setup.cfg is a false positive indicator for pytest

**ID:** BUG-014
**Severity:** low
**Status:** open
**Workspace:** brownfield-repo-analyzer

## Description

Step 2c lists `setup.cfg` as a test runner config file to detect. However, `setup.cfg` is a general Python package configuration file (used for `setuptools`, `mypy`, `flake8`, etc.) that can exist without any `[tool:pytest]` section. Its presence does not reliably indicate that pytest is in use.

This can cause a false ambiguity: if both `setup.cfg` and `pytest.ini` are found, the ambiguity resolution step may fire and ask the user to pick between them, when in reality `setup.cfg` is not a pytest config.

## Fix

Remove `setup.cfg` from the test runner detection list in Step 2c. Add `conftest.py` at root as a more reliable pytest signal (its presence strongly indicates a pytest project).

## Impact

Low — could produce a spurious ambiguity prompt in Python projects that use `setup.cfg` for non-test configuration.

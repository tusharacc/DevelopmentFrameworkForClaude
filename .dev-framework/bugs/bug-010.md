## Description
After removing the `^` anchor fix in BUG-008, `hooks/check-phase.sh` line 55 now matches any path containing `hooks/` or `skills/`:
```bash
if echo "$FILE_PATH" | grep -qE "CLAUDE\.md|hooks/|skills/|agents/|\.claude"; then
```
A file like `/src/cool-hooks/config.py` or `/my-skills/app.js` would incorrectly pass through the phase gate without restriction.

## Expected Behaviour
Only files within the framework's own `hooks/`, `skills/`, `agents/` directories should be allowlisted — not any file with those strings in the path.

## Fix Ideas
- Use a more specific pattern that anchors to the project root or a known path segment, e.g. `/DevelopmentFrameworkForClaude/hooks/`
- Or check that the path ends with a known framework extension pattern

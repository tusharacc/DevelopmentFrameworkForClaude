# Artifact Generator Skill

Generates markdown artifact templates for each phase of the development workflow. Each template includes structure, placeholders, and guidance.

## Template Functions

### 1. PO Requirements Template

```bash
generate_po_artifact() {
  local name=$1
  local timestamp=$(date -u +"%Y-%m-%dT%H:%M:%SZ")
  
  cat > ".dev-framework/artifacts/${name}.po.md" << 'EOF'
---
phase: requirements
role: po
workspace: {{WORKSPACE_NAME}}
timestamp: {{TIMESTAMP}}
status: draft
---

# Requirements: {{FEATURE_NAME}}

## Problem Statement
What problem are we solving? What is the user need?

## User Story
As a [user type], I want to [action], so that [benefit].

## Business Context
- Why now? Why is this valuable?
- Who are the stakeholders?
- Success metrics

## Scope

### In Scope
- Feature 1
- Feature 2

### Out of Scope
- Feature 3
- Constraint 1

## Acceptance Criteria

### Functional
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

### Non-Functional
- [ ] Performance requirement
- [ ] Security requirement
- [ ] Accessibility requirement

### Edge Cases
- Edge case 1
- Edge case 2

## Dependencies & Constraints
- External dependencies
- Time constraints
- Resource constraints
- Technical constraints

## Open Questions
1. Question 1?
2. Question 2?

## Approvals
- Product Owner: [pending]
- Stakeholders: [pending]

---
*This artifact was generated by the PO agent. Last updated: {{TIMESTAMP}}*
EOF

  echo ".dev-framework/artifacts/${name}.po.md"
}
```

## Master Generator Function

```bash
generate_all_artifacts() {
  local name=$1
  
  generate_po_artifact "$name"
  generate_architect_artifact "$name"
  generate_developer_artifact "$name"
  generate_reviewer_artifact "$name"
  generate_tester_artifact "$name"
  generate_observer_artifact "$name"
  
  echo "All artifacts generated for: $name"
}
```

## Usage

Called by command implementations when advancing phases:
- **new-feature** → generates PO artifact
- **hand-off** → generates next phase artifact
- **initialize workspace** → optionally generates all templates

## Template Customization

Templates use `{{PLACEHOLDER}}` syntax for:
- `{{WORKSPACE_NAME}}` - Feature/bug name
- `{{FEATURE_NAME}}` - Display name
- `{{TIMESTAMP}}` - ISO 8601 timestamp

These are replaced when the artifact is generated.
